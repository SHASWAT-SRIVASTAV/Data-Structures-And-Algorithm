## ⚡ Overview

Recursion is a core programming paradigm where a function calls itself to solve progressively smaller instances of a problem.

> “To understand recursion, you must first understand recursion.”

---

## 🧠 Core Concepts

* 🔹 Base Case → Terminates recursion
* 🔹 Recursive Case → Self-calling logic
* 🔹 Call Stack → Tracks function execution

---

## 🧩 Implementations

### 🔁 Fundamental Patterns

* Head Recursion
* Tail Recursion
* Head + Tail Recursion
* Tree Recursion
* Indirect Recursion

---

### 🧮 Mathematical Applications

* Sum of N Numbers
* Factorial
* Power (Linear & Fast Exponentiation)
* Taylor Series (Standard & Horner’s Rule)
* nCr (Factorial & Pascal’s Triangle)

---

### 🏗️ Classic Problem

* Tower of Hanoi

---

## ⏱️ Complexity Insights

| Type                | Complexity |
| ------------------- | ---------- |
| Linear Recursion    | O(n)       |
| Tree Recursion      | O(2ⁿ)      |
| Optimized Recursion | O(log n)   |

---

## 💡 Key Takeaways

* Every recursive solution must have a **base case**
* Recursion uses additional **stack memory**
* Many recursive problems can be optimized using:

  * Memoization
  * Iteration
  * Divide & Conquer
